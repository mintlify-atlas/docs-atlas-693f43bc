Scada-LTS is an open-source, web-based Supervisory Control and Data Acquisition (SCADA) system designed for industrial environments. Built as a community-driven successor to ScadaBR, it provides a full-featured platform for connecting to industrial devices, collecting real-time data, visualizing process states, and responding to alarms — all through a browser-based interface that runs on any modern operating system.

## What Scada-LTS does

Industrial facilities rely on SCADA systems to monitor equipment, collect sensor readings, detect fault conditions, and generate historical reports. Scada-LTS delivers all of these capabilities in a single deployable application. It supports a wide range of industrial communication protocols out of the box, making it possible to connect to PLCs, RTUs, sensors, and other field devices without proprietary middleware.

The system's web-based architecture means operators can monitor and control processes from any networked workstation or device without installing dedicated client software. Administrators manage the system through the same interface, configuring data sources, defining alarm conditions, and controlling user access.

## Architecture

Scada-LTS is a Java web application packaged as a WAR file and deployed on Apache Tomcat. The backend is written in Java 11 and handles all data acquisition, event processing, alarm management, and API serving. The frontend is a Vue.js single-page application bundled into the WAR at build time, communicating with the backend through a combination of DWR (Direct Web Remoting) endpoints and REST API calls.

Data is persisted in a MySQL 8 database. The JDBC connection pool is configured in Tomcat's `context.xml`, with a named resource (`jdbc/scadalts`) that the application looks up at startup. In production Docker deployments, the database container and the application container are connected through Docker networking, with the application waiting for the database to become available before starting Tomcat.

| Layer | Technology |
|---|---|
| Application server | Apache Tomcat 9.0 |
| Backend language | Java 11 |
| Frontend framework | Vue.js |
| Database | MySQL 8 |
| Build tool | Gradle 6.8.1 |
| Container runtime | Docker / Docker Compose |
