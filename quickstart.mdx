title: "Getting Started with Scada-LTS Using Docker Compose"

The fastest way to run Scada-LTS is with Docker Compose. The official `docker-compose.yml` starts three services — a MySQL 8 database, the Scada-LTS application on Tomcat 9, and a HiveMQ MQTT broker — and wires them together automatically. You do not need to install Java, Tomcat, or MySQL on your host machine.

## Prerequisites

- [Docker Engine](https://docs.docker.com/engine/install/) 20.10 or later
- [Docker Compose](https://docs.docker.com/compose/install/) v2 (the `docker compose` plugin, not the standalone `docker-compose` binary)
- Ports `8080`, `3306`, `1883`, and `8081` available on your machine

<Info>
  The Scada-LTS application container is configured with `-Xmx2G -Xms2G` JVM heap settings. Make sure Docker has at least **3 GB of memory** allocated to containers before starting.
</Info>

## Steps

<Steps>
  <Step title="Create the docker-compose.yml file">
    Create a new directory for your Scada-LTS deployment and save the following file as `docker-compose.yml` inside it. This is the exact content from the official repository.

    ```yaml docker-compose.yml
    # Docker compose file to manage your deployed images.
    # Use MySQL server 8 and latest Scada-LTS local build.
    version: '3'
    services:
        database:
            container_name: mysql
            image: mysql/mysql-server:8.0.32
            ports:
                - "3306:3306"
            environment:
                - MYSQL_ROOT_PASSWORD=root
                - MYSQL_USER=root
                - MYSQL_PASSWORD=root
                - MYSQL_DATABASE=scadalts
            expose: ["3306"]
            volumes:
                - ./db_data:/var/lib/mysql:rw
                - ./db_conf:/etc/mysql:ro
            command: --log_bin_trust_function_creators=1
        scadalts:
            image: scadalts/scadalts:latest
            environment:
                - CATALINA_OPTS=-Xmx2G -Xms2G
            ports:
                - "8080:8080"
            depends_on:
                - database
            expose: ["8080", "8000"]
            volumes:
                - ./tomcat_log:/usr/local/tomcat/logs:rw
            links:
                - database:database
            command:
                - /usr/bin/wait-for-it
                - --host=database
                - --port=3306
                - --timeout=30
                - --strict
                - --
                - /usr/local/tomcat/bin/catalina.sh
                - run
        mqtt:
            image: hivemq/hivemq4:latest
            ports:
                - "8081:8080"
                - "1883:1883"
            expose: [ "8081", "1883" ]
    ```

    The `scadalts` service uses `wait-for-it` to delay Tomcat startup until MySQL is accepting connections on port 3306. Database data is persisted in a `./db_data` volume on your host so it survives container restarts.
  </Step>

  <Step title="Start the services">
    From the directory containing your `docker-compose.yml`, run:

    ```bash
    docker compose up -d
    ```

    Docker pulls the `scadalts/scadalts:latest`, `mysql/mysql-server:8.0.32`, and `hivemq/hivemq4:latest` images on first run. Subsequent starts are faster.

    Watch the application log to confirm Tomcat has started:

    ```bash
    docker compose logs -f scadalts
    ```

    Look for a line containing `Server startup in` followed by a time in milliseconds. This indicates Tomcat is ready.

    <Note>
      First startup takes longer because MySQL initialises the `scadalts` database schema. Allow up to 60 seconds before the application becomes available.
    </Note>
  </Step>

  <Step title="Open the web interface">
    Once Tomcat has started, open your browser and navigate to:

    ```
    http://localhost:8080/Scada-LTS
    ```

    You should see the Scada-LTS login page.

    <Tip>
      The context path is `/Scada-LTS` (capital S and capital LTS). Navigating to `http://localhost:8080/` returns a 404 because no root application is deployed.
    </Tip>
  </Step>

  <Step title="Log in with default credentials">
    Use the following credentials on your first login:

    | Field | Value |
    |---|---|
    | Username | `admin` |
    | Password | `admin` |

    <Warning>
      Change the default admin password immediately after your first login. Navigate to **Administration → Users**, select the `admin` user, and set a strong password before exposing the instance to a network.
    </Warning>
  </Step>

  <Step title="Add your first data source">
    A Virtual data source lets you verify the system is working without any physical hardware.

    1. In the top navigation, click **Data Sources**.
    2. Click **Add data source** and select **Virtual data source** from the type list.
    3. Enter a name such as `Test Virtual Source` and click **Save**.
    4. With the data source open, click **Add data point**.
    5. Choose **Numeric** as the point type and set the update period to `5` seconds.
    6. Enter a name such as `Temperature Sim` and click **Save**.
    7. Enable the data source by clicking the toggle next to its name.

    The data point now appears in the watchlist and begins generating simulated numeric values every five seconds. You can add it to a graphical view or create an alarm detector to explore other features.
  </Step>
</Steps>

## MQTT broker

The `mqtt` service in the Compose file runs [HiveMQ 4](https://www.hivemq.com/hivemq/mqtt-broker/), a full-featured MQTT broker. It listens on:

- `1883` — standard MQTT (unencrypted)
- `8081` — HiveMQ web management console

To connect an MQTT data source in Scada-LTS to the local broker, use `mqtt` as the hostname (Docker's internal DNS resolves it within the Compose network) and `1883` as the port. From your host machine, use `localhost:1883`.

## Next steps

This quickstart uses the pre-built `scadalts/scadalts:latest` Docker image, which is suitable for evaluation and small deployments. For production use, custom builds from source, or WAR-based deployments on an existing Tomcat server, see the full [installation guide](/installation).
